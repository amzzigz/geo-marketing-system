## TASK-003: 用户注册与登录
- **类型**: feature
- **模块**: users
- **描述**:
  - POST /api/users/register（用户名+密码+手机号）
  - POST /api/users/login（返回JWT token）
  - GET /api/users/me（当前用户信息）
  - JWT中间件：解析token、注入当前用户
  - 密码 bcrypt 加密
  - Pydantic Schema 定义请求/响应
- **验收标准**: 能注册→登录→拿token→访问/me，无token返回401
- **依赖**: TASK-001, TASK-002
- **状态**: ⬜

## TASK-004: 企业资料管理
- **类型**: feature
- **模块**: knowledge
- **描述**:
  - POST /api/knowledge/upload（上传docx/txt/pdf，存本地uploads/knowledge/）
  - GET /api/knowledge（当前用户文件列表，分页）
  - GET /api/knowledge/{id}（文件详情含全文）

- **验收标准**: 上传一个docx，能看到提取的文本和分段列表
- **依赖**: TASK-003
- **状态**: ⬜

## TASK-005: 图片素材管理
- **类型**: feature
- **模块**: images
- **描述**:

- **验收标准**: 创建分类→上传图片→列表能看到缩略信息
- **依赖**: TASK-003
- **状态**: ⬜

## TASK-008: 手动生成文章
- **类型**: feature
- **模块**: articles
- **描述**:
  - POST /api/articles/generate 参数：
    - core_keyword_id（必填）
    - expansion_keywords[]（可选，手动输入或从拓展词选）
    - title_prompt_id（标题指令）
    - content_prompt_id（内容指令）
    - knowledge_file_ids[]（企业资料）
    - image_category_id（图片分类）
    - count（生成数量，1-10）
  - 后端逻辑：
    - 拼装prompt = 指令模板 + 企业资料摘要 + 关键词
    - 调用 One API 生成标题和正文
    - 随机插入图片URL到正文
    - 存入 mkt_articles，状态=review
  - GET /api/articles（文章列表，分页，按状态筛选）
  - GET /api/articles/{id}（文章详情）
  - PUT /api/articles/{id}（编辑标题/正文）
  - One API调用封装为独立service，超时60s，重试3次
- **验收标准**: 选择主词+指令+资料→点生成→能看到AI返回的文章，图片已插入
- **依赖**: TASK-004, TASK-005, TASK-006, TASK-007
- **状态**: ⬜
